# Quickstart

Get from zero to a real, runnable Godot 4 game — generated by your own local model, with real
art, natural-language edits, and one-command publishing. For the bigger picture see `README.md`
and `WHY.md`; for the full command list, `playsmith --help`.

---

## 1. Prerequisites

Required:

1. **Python 3.11+** (`python3 --version`).
2. **Godot 4.x** — the standard build. Confirm `godot --version` prints `4.x`. If `godot` isn't
   on your `PATH`, set the full path in `config/playsmith.yaml` (`engine.godot.binary`). For
   HTML5 export you also need the export templates (Godot editor → *Editor → Manage Export
   Templates*).
3. **A model.** Easiest is a local one via [Ollama](https://ollama.com):
   ```bash
   ollama pull qwen2.5-coder:7b      # modest hardware
   ollama pull qwen3-coder           # stronger machine: larger, agentic-tuned, big context
   ```
   Ollama serves an OpenAI-compatible endpoint at `http://localhost:11434/v1`. Any
   OpenAI-compatible server works (LM Studio, vLLM, llama.cpp) — or a cloud key (see §7).

Optional (everything still runs without these — you just get placeholders / no publish):

- **ComfyUI** at `http://localhost:8188` for AI sprite generation.
- **butler** (itch.io CLI) for publishing.

> ⚠️ **Context window matters.** Local models need a large context for agentic editing. Keep
> `num_ctx` at 16K–32K (default 16384). The 4K default breaks file editing.

---

## 2. Install

```bash
git clone https://github.com/raihan-js/playsmith && cd playsmith
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"      # drop [dev] if you don't want ruff/pytest
```

---

## 3. Configure

```bash
cp config/playsmith.example.yaml config/playsmith.yaml
playsmith config-check          # shows how your config resolves (providers, routes, fallback)
```

Edit `config/playsmith.yaml` to point at your model and Godot binary, and to choose where games
are written (`workspace_dir`, default `~/playsmith-games`).

---

## 4. Verify the foundations

Each proves one pillar before you generate a game:

```bash
playsmith models          # shows the route table + round-trips a message to your model
playsmith engine-check    # scaffolds a trivial Godot project and runs it headless
playsmith skills          # lists the genres you can make (2d-platformer, visual-novel)
```

If `models` fails, your model server isn't reachable. If `engine-check` fails, Godot isn't found
or isn't 4.x.

---

## 5. Make a game

```bash
playsmith new "a 2D platformer where a cat collects fish and avoids spikes" --yes
# or a story game:
playsmith new "a short branching mystery about a lighthouse keeper" --yes
```

What happens: the prompt is **routed** to a skill (`2d-platformer` or `visual-novel`), a real
Godot 4 project is **scaffolded**, the agent **writes** scenes and GDScript, then **runs** and
**verifies** it — checking real gameplay assertions headless (`player_on_floor`, `scene_loads`,
`no_errors`, …) and fixing until they pass. Drop `--yes` to approve each file write yourself
(you'll see a diff).

---

## 6. Art, edit, play, export, publish

```bash
# Bring your own art (no GPU needed):
playsmith assets import ~/Pictures/cat.png
# ...or generate it (needs ComfyUI running):
playsmith assets generate "a pixel-art cat" --kind sprite

# Iterate in natural language (re-runs + re-verifies):
playsmith edit "make the player jump higher and add a second platform"

# Play / open / export:
playsmith run                                       # play the latest game in a window
godot --editor --path ~/playsmith-games/<your-game> # open & edit by hand
playsmith export --target web                       # headless HTML5 export to <project>/build/

# Publish to itch.io (needs butler configured):
playsmith publish --itch you/your-game
```

The generated project is **yours** — real, editable Godot source, no lock-in.

---

## 7. Use a cloud model (optional) + the router

Everything runs locally by default. To use a cloud key, or to lean on a stronger model for the
hard steps, configure `llm.routes` and/or `llm.fallback` in `config/playsmith.yaml` (see the
commented examples there). The router picks a provider per task; whenever it crosses from a
**local** model to a **cloud** one it **warns you** (your code is leaving the machine).

> **Anthropic note:** use `kind: "anthropic"` (native `/v1/messages`). Anthropic's
> OpenAI-compatible endpoint drops strict tool schemas and prompt caching, which is unreliable
> for the agent's tool loop. Other providers (OpenAI, OpenRouter, Gemini-compat) use the default
> `kind: "openai"`.

`playsmith models` prints the full route table so you can see what each task resolves to.

## 8. Going further (Phase 2): 3D, community skills, Unreal

```bash
# 3D games (Godot) — same flow, just ask for 3D:
playsmith new "a simple 3D platformer where a robot collects orbs"
playsmith assets generate "a low-poly tree" --kind mesh   # needs a mesh backend; else primitives

# Community skills (marketplace) — installs are integrity-checked and never auto-run:
playsmith skills search rpg
playsmith skills install <name>                # untrusted skills need --allow-untrusted
playsmith skills remove <name>

# Router maturity — measure tool-call reliability and get a fallback recommendation:
playsmith models --eval

# Unreal track (EXPERIMENTAL; Godot is the default, tested engine):
playsmith unreal royalty 2000000               # Epic charges 5% above $1M/product; Godot charges 0
playsmith unreal check                         # editor + Remote Control availability
```

> Community skills are *code you're trusting* — their scripts end up in your game and their
> instructions drive the agent. Playsmith verifies a checksum, refuses untrusted skills unless you
> opt in, never executes anything on install, and shows you every file diff. Prefer the curated index.

## Troubleshooting

- **Model is flaky / gives up mid-build.** Raise `num_ctx` (16K → 32K), use a stronger model
  (`qwen3-coder`), or set a cloud `llm.fallback` for CODING/REASONING. See `BUILD_PLAN.md` →
  "If a local model struggles."
- **Assertions fail (e.g. `player_on_floor`).** The agent normally self-corrects; if it gives
  up, `playsmith edit "the player falls through the floor — give the ground a collision shape"`.
- **ComfyUI / butler not found.** Both are optional — games still generate with placeholders and
  you can export locally. Install them only when you want generated art / itch publishing.
- **Export fails.** Install the HTML5 export templates in the Godot editor.
- **Screenshots are blank.** `--headless` can't render pixels; verification is assertion-based
  (which works headless). For a real screenshot use a desktop or `xvfb-run`.
- **"No generated project found."** Run `playsmith new "..."` first; `edit`/`run`/`export`/
  `publish` default to the most recent project (or pass `--project <dir>`).
